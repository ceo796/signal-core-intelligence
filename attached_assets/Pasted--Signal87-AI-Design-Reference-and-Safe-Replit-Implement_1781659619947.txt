## Recommended design direction
Use the uploaded files as the design source of truth for a more cohesive Signal87 platform.

The strongest visual direction is:
- Cream / warm off-white internal app surfaces
- Deep ink / near-black typography
- Minimalist left navigation
- Large centered intelligence prompt areas
- Thin borders and restrained card treatment
- Elegant, sparse but not empty layout density
- Abstract signal/radar brand language from the circular mark
- Optional dark mode experience based on `product-home-dark.png`
- Premium enterprise feel, not consumer SaaS clutter

## Key design inconsistencies to clean up
1. Logo inconsistency
   - Current landing screenshots use a purple angular/mountain mark.
   - Uploaded design files use the circular radar/signal mark.
   - Choose one identity system and apply consistently across landing, app, auth, nav, favicon, and loading states.

2. Typography inconsistency
   - Current landing page text is under-sized and too widely spaced.
   - Uploaded app screens use stronger hierarchy and a more editorial feel.
   - Increase hero typography, page titles, and card titles.

3. Spacing inconsistency
   - Current landing page has excessive white space and weak visual density.
   - Uploaded files use centered compositions with more purposeful spacing.
   - Reduce vertical gaps, increase content scale, and align sections to a stronger grid.

4. Card and border inconsistency
   - Standardize border radius, border color, card padding, hover states, and shadows.
   - Prefer thin warm borders and very subtle elevation.

5. Motion inconsistency
   - Add restrained motion only where it improves perceived quality.
   - Use hover lift, soft border glow, cursor-reactive background accents, and subtle parallax on hero visuals.
   - Respect `prefers-reduced-motion`.

## Safe Replit prompt

```text
Use the uploaded `Signal87 AI Design System (20).zip` as the visual reference package for a frontend-only design consistency sweep.

First, unpack and review the zip. Treat these files as design references:
- product-documents.png
- product-home-dark.png
- product-intelligence-empty.png
- product-intelligence-query.png
- product-intelligence-typed.png
- reference-rebrand.png
- reference-screenshot.png
- all Signal87 logo/mark SVG and PNG assets

Goal:
Clean up visual inconsistencies across the Signal87 platform and align the product with the uploaded design system.

Strict scope:
This is frontend design cleanup only.
Do not change backend logic, authentication, routing, document upload logic, chat logic, brief generation, APIs, database queries, environment variables, Clerk/Supabase/OpenAI configuration, protected routes, or production platform functionality.

Design direction:
- Use a premium enterprise-grade minimalist style.
- Use cream / warm off-white backgrounds for internal app surfaces where appropriate.
- Use deep ink typography.
- Use restrained violet accents only where they support hierarchy or interaction.
- Use the uploaded Signal87 circular radar/signal mark consistently unless there is an existing production reason not to.
- Reduce excessive whitespace.
- Increase typography scale and visual density.
- Standardize cards, borders, buttons, icons, and spacing.
- Preserve the clean white landing aesthetic but make it stronger, larger, more animated, and less sparse.

Motion direction:
Create a premium, enterprise-grade interactive feel similar to a cursor-responsive WebGL or parallax hero animation, but keep it restrained and lightweight.
- Add subtle cursor-reactive / mouse-tracking effects only in safe visual areas.
- Add soft animated signal lines, radar rings, document nodes, or particle accents where appropriate.
- Add restrained card hover lift, border glow, and button hover polish.
- Respect `prefers-reduced-motion`.
- On mobile, disable cursor-based effects and use simple touch-safe transitions.
- Avoid heavy WebGL unless isolated and performance-safe.

Pages to review and align:
- Landing page
- Sign in / sign up / pending access pages
- Dashboard / home
- Document library
- Document detail page
- Intelligence / chat page
- Brief generator
- Upload flow
- Empty states
- Loading states
- Error states
- Mobile and tablet layouts

Implementation rules:
1. Start with an audit only. Create `DESIGN_QA_REPORT.md` before changing code.
2. Identify exact files/components involved.
3. Recommend a small phased implementation plan.
4. Stop and ask for approval before applying changes.
5. After approval, make small reversible frontend-only changes.
6. Prefer shared tokens, CSS variables, Tailwind theme updates, and reusable component cleanup.
7. Do not introduce major dependencies unless already present.
8. Run typecheck/build and test main routes after changes.

Deliverable for audit phase:
Create `DESIGN_QA_REPORT.md` with:
- Executive summary
- Current design inconsistencies
- Comparison against uploaded design system
- Priority ranking: Critical / High / Medium / Low
- Page-by-page findings
- Exact recommended fixes
- Files likely involved
- Risk areas to avoid
- Implementation plan in small safe phases

Stop after the report and ask for approval.
```

## Recommended implementation phases

### Phase 1: Identity and tokens
- Standardize logo usage.
- Add shared color, spacing, radius, typography, and shadow tokens.
- Do not alter page logic.

### Phase 2: App shell consistency
- Align sidebar, top nav, content width, spacing, cards, and page headers.
- Apply cream/ink internal app aesthetic.

### Phase 3: Intelligence and document pages
- Align prompt area, empty states, query cards, document cards, and response states with uploaded references.

### Phase 4: Landing page density and animation
- Increase hero typography.
- Reduce dead space.
- Add lightweight cursor-reactive abstract signal motion.
- Add subtle animated radar/signal visuals.

### Phase 5: Responsive QA
- Test mobile, tablet, and desktop.
- Disable cursor effects on mobile.
- Confirm no protected functionality was changed.
```
